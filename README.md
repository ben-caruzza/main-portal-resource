# main-portal-resource

## 项目简介

`main-portal-resource` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态资源的集中存储与版本管理。

这里存放的页面彼此独立，可用于展示、测试、参考或作为其他项目的前端素材。每个页面均以独立文件形式存在，便于直接访问或嵌入使用。

## 目录结构

```
main-portal-resource/
├── pages/          # 存放所有 HTML 页面文件
│   ├── example-1.html
│   ├── example-2.html
│   └── ...
├── assets/         # 页面引用的静态资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
├── docs/           # 相关文档或说明（可选）
└── README.md       # 本文件
```

- **pages/**：所有独立 HTML 页面的主目录。每个页面应自包含或引用 `assets/` 中的资源。
- **assets/**：存放页面共用的或特定的静态文件，按类型组织。
- **docs/**：可选的附加文档、设计稿或使用说明。

## 页面归档说明

- 每个 HTML 页面均为独立文件，不依赖外部服务或特定域名。
- 页面之间无耦合，可单独部署或通过 GitHub Pages 等静态托管服务访问。
- 归档的页面可能包含示例、原型、工具页面或临时展示内容，请根据实际需要查阅。
- 页面文件名应具有描述性，便于识别内容；如需版本管理，可在文件名中加入日期或版本号。

## 使用方式

1. **直接浏览**：在仓库中点击 `pages/` 目录下的 HTML 文件，选择“View raw”或通过 GitHub Pages 访问（如已启用）。
2. **本地运行**：将仓库克隆到本地，用浏览器打开 `pages/` 中的 HTML 文件即可。
3. **引用资源**：如需在外部项目中使用本仓库的静态资源，可通过原始文件链接（raw URL）引用，或自行下载整合。

## 维护说明

- 新增页面：请将 HTML 文件放入 `pages/` 目录，如有依赖资源请一并放入 `assets/` 对应子目录。建议在提交信息中简要说明页面用途。
- 更新页面：直接修改对应文件，并确保不破坏与其他页面的兼容性（如有共享资源）。
- 删除页面：若页面不再需要，请移除相关文件及资源，并更新本 README 中的目录结构描述（如有必要）。
- 本仓库不自动部署，提交后需手动或通过 CI 触发静态站点更新（如配置了 GitHub Pages）。
